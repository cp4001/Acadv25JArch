# T3 커맨드 사양서 — colinear L1·L3 을 L2 교차점으로 정리 (TrimColinearToCross)

- **문서 버전**: v1.0 (구현 완료 기준 정리)
- **작성일**: 2026-07-09
- **커맨드명**: `T3`
- **메서드명**: `T3_TrimColinearToCross`
- **네임스페이스**: PipeLoad2 (`CommandUtil` 클래스, `Util_Command.cs`)
- **대상 환경**: AutoCAD 2025 (.NET 8.0), Windows 11
- **시스템**: 신규 시스템 — 외부 유틸 의존 없음
- **상태**: 구현 완료 (2026-07-08 커밋 "T3 T Line combine")

---

## 1. 개요

Line 3개(동일선상 2개 + 각도를 가진 1개)를 선택하면, 어느 것이 "동일선상 쌍"이고
어느 것이 "각도 있는 선"인지 자동으로 판별한 뒤, 셋의 가까운 끝점을 전부
동일선상 쌍과 각도선의 실제 교차점으로 이동시켜 T자 접합부를 깔끔하게 정리한다.

- 동일선상 쌍을 `L1`, `L3` 로, 나머지 하나를 `L2` 로 부른다.
- `L1`·`L2` 교차점을 구해 `L1`, `L2`, `L3` 세 Line 모두 그 점으로 끝점을 이동(연장 또는 자르기).

---

## 2. 명령 절차

1. `LINE` 필터로 정확히 3개 Line 선택(3개가 아니면 오류 메시지 후 중단)
2. **colinear 쌍 자동 판별**: 3개 중 2개씩 짝지어 총 3쌍을 모두 검사(`IsColinearT3`) —
   평행(각도 오차 이내) + 상대 끝점의 수직 오프셋이 오차 이내인 쌍을 colinear 로 판정
   - colinear 쌍이 정확히 1개가 아니면("0개" 또는 "2개 이상") 오류 메시지 후 중단
   - colinear 로 판정된 두 Line = `L1`, `L3` / 나머지 하나 = `L2`
3. `L1`·`L2` 교차점(`IntersectWith`, `Intersect.ExtendBoth` — 양방향 무한 연장) 계산
   - 교차점이 정확히 1개가 아니면(평행 등) 오류 메시지 후 중단
4. `L1`, `L2`, `L3` 각각의 **교차점에 더 가까운 쪽 끝점**을 교차점으로 이동(`MoveNearEndT3`)
   — 기존 끝점보다 교차점이 멀면 연장, 가까우면 자르기 효과
5. 결과 메시지 출력

---

## 3. 판정 조건

### 3.1 Colinear 판정 (`IsColinearT3`)

- 평행 판정: `IsParallelScl`(SCL 의 헬퍼를 재사용) — 방향 반대 포함, 각도 오차 `T3_ANGLE_TOL`(1.0°) 이내
- 오프셋 판정: 한 Line 의 양 끝점을 상대 Line 의 무한직선에 수직 투영(`GetClosestPointTo(pt, true)`)했을 때
  거리가 `T3_OFFSET_TOL`(1.0) 이내면 동일 직선상으로 판정
- 두 조건 모두 만족해야 colinear

### 3.2 끝점 이동 (`MoveNearEndT3`)

- `line.StartPoint`/`EndPoint` 중 대상 점(target)에 더 가까운 쪽만 `target`으로 치환
- `IsWriteEnabled`가 아니면 `UpgradeOpen()` 후 이동

---

## 4. 상수 요약

| 상수 | 값 | 의미 |
|---|---|---|
| `T3_ANGLE_TOL` | 1.0 | colinear 각도 허용 오차(도) |
| `T3_OFFSET_TOL` | 1.0 | colinear 수직 투영 오프셋 허용 거리 |

- 평행 판정 로직(`IsParallelScl`)은 `SCL` 리전의 헬퍼를 그대로 재사용(중복 구현 없음)

---

## 5. 사용 API

| API | Return 유형 | 용도 |
|---|---|---|
| `Editor.GetSelection(PromptSelectionOptions, SelectionFilter)` | `PromptSelectionResult` | Line 3개 선택 |
| `Curve.GetClosestPointTo(Point3d, bool)` | `Point3d` | colinear 오프셋 판정(수직 투영, `extend=true`) |
| `Entity.IntersectWith(Entity, Intersect, Point3dCollection, IntPtr, IntPtr)` | `void` | L1·L2 교차점(`ExtendBoth`) |
| `Line.StartPoint / EndPoint` (set 가능) | `Point3d` | 끝점 이동(연장/자르기) |
| `DBObject.IsWriteEnabled` / `UpgradeOpen()` | `bool`/`void` | 쓰기 모드 전환 |

---

## 6. 미확정/확인 필요 사항

1. **colinear 쌍이 2개 이상 검출되는 경우**(예: 세 Line 이 모두 한 직선 위에 있는 경우) —
   현재는 "colinearPairCount != 1"로 일괄 오류 처리되는데, 이 케이스를 별도로 안내할지 확인 필요.
2. Base 라인 XData(예: Duct/Pipe 계열)가 새로 이동된 끝점에도 그대로 유지되는지 — 좌표만 옮기고
   XData/속성은 건드리지 않으므로 원래 값 그대로 유지될 것으로 예상(코드상 별도 XData 처리 없음).
